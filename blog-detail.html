How I Built My Portfolio

Notes from designing and shipping this site — what worked, what didn't.

Building a personal portfolio is, in many ways, the most personal kind of design work. It's the one project where you are both the client and the designer, and the brief is wide open.

Starting from intent

Before sketching any layouts, I wrote down the feeling I wanted: quiet confidence. Not flashy, not minimal-to-the-point-of-empty — just calm and honest.

Choosing the stack

HTML, CSS, JavaScript. No framework, no backend. The site is meant to last and be easy to host anywhere.

Why no framework?

Because the content is simple, and a framework would have added build complexity without meaningful benefits.

Design system first

I defined CSS variables for color, spacing, and radius before writing any pages. That single step paid off many times over.

What's next

I'll keep iterating — adding case studies, improving accessibility, and writing more notes like this one.
